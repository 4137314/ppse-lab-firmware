Detailed Description

Gestione dell'input utente, sitema di debounce e macchina a stati del menu.

  • Questo modulo implementa la logica di navigazione tramite pulsanti GPIO, utilizzando interrupt (IRQ) e allarmi hardware per il debounce. Gestisce inoltre la logica di timeout del display e le combo di tasti.

◆ debounce_callback()

long long debounce_callback ( alarm_id_t  id,
void *  user_data 
)

Callback dell'allarme per il filtraggio software del debounce.

Funzione richiamata dall'allarme hardware per confermare la pressione (Debounce).

  • Verifica se dopo DEBOUNCE_DELAY il pin è ancora basso. In caso positivo, valida la pressione impostando il relativo flag _press.
  • Parameters
    idID dell'allarme hardware.
    user_dataPuntatore al numero del pin GPIO passato dall'handler.
    Returns
    long long 0 per indicare che l'allarme non deve ripetersi.

Handler degli interrupt GPIO per i pulsanti.

  • Gestisce i fronti di discesa (pressione) e di salita (rilascio). Al rilevamento della pressione, disabilita l'interrupt e avvia un allarme di debounce.
